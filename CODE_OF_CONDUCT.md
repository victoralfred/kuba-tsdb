# Kuba TSDB Code of Conduct

## Our Commitment

Kuba TSDB is built on principles of technical excellence and collaborative engineering. We are committed to maintaining a community where contributors can focus on what matters most: building high-performance, reliable time-series database software.

## Core Values

### Technical Excellence
- We prioritize correctness, performance, and reliability
- Code quality and thorough testing are non-negotiable
- We value data integrity and system stability above convenience

### Constructive Collaboration
- Technical disagreements are resolved through evidence, benchmarks, and reasoned discussion
- We review code critically but respectfully
- We help others improve through clear, actionable feedback

### Professional Conduct
- We communicate directly and professionally
- We assume good intent in technical discussions
- We acknowledge mistakes and learn from them

## Expected Behavior

- **Be direct**: Provide clear, honest technical feedback
- **Be helpful**: Share knowledge and assist others in understanding the codebase
- **Be thorough**: Write tests, document your changes, and consider edge cases
- **Be respectful**: Treat all contributors with professional courtesy
- **Be patient**: Remember that contributors have varying experience levels

## Unacceptable Behavior

- Personal attacks or harassment of any kind
- Discriminatory language or conduct
- Publishing others' private information without permission
- Deliberately introducing bugs, security vulnerabilities, or malicious code
- Sustained disruption of project discussions or processes

## Scope

This Code of Conduct applies to:
- GitHub issues, pull requests, and discussions
- Project communication channels
- Project-related events or meetups

## Enforcement

Instances of unacceptable behavior may be reported to the project maintainers at [voseghale1@gmail.com] or by opening a private issue. All reports will be reviewed and handled appropriately.

Project maintainers reserve the right to remove, edit, or reject contributions that do not align with this Code of Conduct, and to temporarily or permanently ban contributors for behavior deemed inappropriate, threatening, or harmful.

## Attribution

This Code of Conduct is tailored for the Kuba TSDB project and its focus on high-performance database engineering.
